# 大规模遥感栅格分层抽样优化器 (GeoSample Optimizer)

本项目提供了一套高效、可扩展的 Python 方案，用于从大规模 TIF 栅格影像（如 ESA WorldCover 10m 数据）中提取统计学公平的分层随机样点。

针对处理 TB 级遥感数据时常见的 I/O 瓶颈和内存溢出问题，本项目将工作流拆分为**分布式提取**与**流式合并**两个阶段，合并速度相比传统矢量操作方案提升 100 倍以上。

## 🚀 核心原理：两阶段加权蓄水池抽样

在海量文件中直接进行全局等概率抽样极具挑战。本项目采用 **两阶段蓄水池抽样 (Two-Stage Reservoir Sampling)** 算法，确保不同大小的 TIF 文件中的像素被抽中的概率绝对公平：

1. **第一阶段 (Map)**：在每个 TIF 文件内部进行独立的蓄水池抽样，并记录该文件有效像素的**总数量（即权重 Weight）**。
2. **第二阶段 (Reduce)**：合并各个文件的临时样本时，进行**加权抽样**。文件原始像素基数越大，其样本在最终池子中被保留的概率越高，从而完美还原全局等概率分布。

## ⚙️ 核心工作流与技术细节

### 阶段一：特征提取 (Map 阶段)
* **多进程并发与锁机制**：利用 `multiprocessing` 开启多个工作进程并行扫描 TIF 文件。通过进程锁 (`Lock`) 控制终端输出，避免多进程打印造成的进度条穿插和乱码。
* **亚像素级空间随机化**：利用 `rasterio` 的仿射变换矩阵获取像素左上角物理坐标后，根据影像分辨率（如 10m）向 X 和 Y 方向加上对应范围的随机偏移量，消除样本的网格化人工痕迹，实现真正的空间随机。
* **极简缓存设计**：提取结果不直接生成矢量文件，而是保存为极小的 `.pkl` 缓存。缓存内部采用**字典 (Dictionary)** 结构，以地类 ID（如 `10` 代表林地）为键，分离存储坐标列表与该地类的权重，极大优化了后续查询效率。

### 阶段二：合并抽样 (Reduce 阶段)
* **单进程极速合并**：由于第一阶段已将庞大影像浓缩为轻量级字典缓存，本阶段采用单进程流式读取，瞬间完成内存级别的加权运算。
* **高性能写入**：依托 `pyogrio` 库（基于 GDAL/OGR），将最终的全局样点飞速写入 `.gpkg` 矢量地图文件中。

## 🛡️ 工程稳定性保障

* **断点续传**：程序在第一阶段处理完每个 TIF 文件后，会生成一个对应的 `.done` 标记文件。若遭遇断电或意外崩溃，重启脚本时将自动跳过已带有 `.done` 标记的文件，无缝恢复进度。
* **内存友好**：全程无需一次性加载全量数据或创建海量中间矢量文件，内存占用始终保持恒定。

确保您的 Python 环境中安装了必要的依赖项：
```bash
pip install -r requirements.txt


**2. 第一阶段：特征提取**

修改 script_stage1_extract.py 中的输入路径，运行：

```python script_stage1_extract.py```


该脚本将并行生成中间缓存文件，并实时打印各文件的地类分布诊断信息。

**3. 第二阶段：最终抽样**

运行 script_stage2_sample.py，根据提示输入目标地类 ID（如 10 代表林地），脚本将迅速生成最终的 .gpkg 矢量文件。
